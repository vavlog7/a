# VALINYTSKYI AGENCY — лендинг консультаций для beauty-мастеров

Готовый mobile-only сайт на двух языках для публикации на GitHub Pages.

## Структура репозитория

```text
/
├── index.html             # Русская версия лендинга
├── ua/
│   ├── index.html         # Українська версія лендингу
│   └── thanks/index.html  # Українська сторінка після оплати
├── thanks/index.html      # Русская страница после оплаты
├── assets/config.js       # ЕДИНСТВЕННЫЙ ФАЙЛ ДЛЯ ССЫЛОК И TELEGRAM
└── .nojekyll
```

## Как работает заявка

1. Клиент выбирает тариф на сайте: «Я всё сама» за 29 € или VIP за 149 €.
2. Заполняет имя, контакт, город и вопрос.
3. После нажатия кнопки сайт отправляет сообщение напрямую от Telegram-бота в вашу группу.
4. Клиент сразу перенаправляется на ссылку оплаты выбранного тарифа в WayForPay.
5. Заявка остаётся в вашей Telegram-группе, даже если клиент не оплатил.

Для тестовой версии GitHub Pages отправка в Telegram реализована без backend: токен бота хранится в `assets/config.js` и виден в публичном коде страницы. Используйте отдельного тестового бота и замените его токен перед рабочим запуском.

## Заполнить перед публикацией

Откройте только файл:

```text
assets/config.js
```

И вставьте:

```js
const SITE_CONFIG = {
  telegram: {
    botToken: 'ЗДЕСЬ_ВСТАВИТЬ_ТОКЕН_TELEGRAM_БОТА',
    chatId: 'ЗДЕСЬ_ВСТАВИТЬ_ID_TELEGRAM_ГРУППЫ'
  },
  payments: {
    ru: {
      basic: 'ЗДЕСЬ_ВСТАВИТЬ_ССЫЛКУ_WAYFORPAY_29_EUR_RU',
      vip: 'ЗДЕСЬ_ВСТАВИТЬ_ССЫЛКУ_WAYFORPAY_149_EUR_RU'
    },
    ua: {
      basic: 'ТУТ_ВСТАВИТИ_ПОСИЛАННЯ_WAYFORPAY_29_EUR_UA',
      vip: 'ТУТ_ВСТАВИТИ_ПОСИЛАННЯ_WAYFORPAY_149_EUR_UA'
    }
  },
  bots: {
    consultation: 'ЗДЕСЬ_ВСТАВИТЬ_ССЫЛКУ_НА_ЧАТ_БОТА_ДЛЯ_КОНСУЛЬТАЦИИ',
    training: 'ЗДЕСЬ_ВСТАВИТЬ_ССЫЛКУ_НА_ЧАТ_БОТА_С_ОБУЧЕНИЕМ'
  }
};
```

## Telegram: что нужно настроить

1. Создайте Telegram-бота и получите его token.
2. Добавьте бота в группу, куда должны приходить заявки.
3. Разрешите боту отправлять сообщения в этой группе.
4. Получите `chat_id` группы и вставьте его в `assets/config.js`. Для супергрупп ID обычно начинается с `-100`.

Пример сообщения, которое поступит в группу:

```text
🔴 Новая заявка с сайта

Язык страницы: RU
Имя: Анна
Контакт: @anna_beauty
Страна / город: Варшава

Тариф: Я всё сама
Стоимость: 29 €

Вопрос клиента:
Хочу запустить рекламу, но боюсь слить бюджет.

Клиент отправлен на оплату: 29 €
```

## WayForPay: ссылки оплаты и страницы после оплаты

Рекомендуется создать четыре платёжные ссылки, чтобы сохранить язык страницы после покупки:

| Язык | Тариф | Цена | После оплаты направить на |
|---|---|---:|---|
| RU | Я всё сама | 29 € | `/thanks/` |
| RU | VIP | 149 € | `/thanks/` |
| UA | Я все сама | 29 € | `/ua/thanks/` |
| UA | VIP | 149 € | `/ua/thanks/` |

На страницах после оплаты стоит кнопка входа в бот обучения. Её ссылка также задаётся один раз в `assets/config.js` через `SITE_CONFIG.bots.training`.

## Публикация на GitHub Pages

1. Создайте новый репозиторий на GitHub.
2. Загрузите содержимое этой папки в корень репозитория.
3. Откройте `Settings → Pages`.
4. Выберите `Deploy from a branch`, ветку `main`, папку `/(root)`.
5. Сохраните настройки.

После публикации страницы будут доступны так:

```text
https://ВАШ-ЛОГИН.github.io/НАЗВАНИЕ-РЕПОЗИТОРИЯ/       # RU
https://ВАШ-ЛОГИН.github.io/НАЗВАНИЕ-РЕПОЗИТОРИЯ/ua/    # UA
```

## Проверка перед запуском рекламы

- Откройте русскую страницу и отправьте тестовую заявку по каждому тарифу.
- Проверьте, что две заявки появились в Telegram-группе.
- Проверьте, что после каждой заявки открывается правильная ссылка оплаты.
- Сделайте то же на украинской версии.
- Проверьте обе страницы после оплаты и кнопку перехода в бот обучения.
